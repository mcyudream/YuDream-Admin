# Agent 工作流完整运行时设计

## 目标

将 Agent 编排器中已展示的全部节点接入真实运行逻辑，并让调试面板能够按实际图路径展示节点输入、输出、分支与错误。

## 运行模型

- 应用层解析 Vue Flow JSON 为节点和边，校验唯一开始节点、可达性和环。
- 执行上下文保存用户输入、附件、图片、每个节点输出和命名变量。
- 节点处理器按 kind 分派；技术能力通过领域网关调用，不直接依赖基础设施实现。
- 条件节点只激活命中的 sourceHandle 分支，未命中路径不执行。
- 节点完成事件携带经过裁剪的真实输入/输出摘要，最终由结束节点选择输出。

## 节点语义

- `start/input/end`：初始化输入、映射变量、选择最终输出。
- `llm/understand`：使用节点选择的模型；理解节点要求 JSON 意图结果。
- `condition`：受限表达式判断并选择 true/false 分支。
- `code/tool`：通过 Python Runtime 执行内联脚本或已授权工具。
- `template`：替换上下文变量生成文本。
- `search/vector`：按知识空间调用完整检索或原始向量检索。
- `rerank/embedding`：调用知识库模型网关处理候选或文本。
- `document`：通过文档文本提取网关读取上传内容。
- `citation`：将检索命中确定性转换为引用列表和 Markdown 来源。

## 接口与 UI

- Agent 元数据接口只返回知识空间和可用模型标识，不返回密钥。
- 检查器为每种节点展示对应配置，并提供明确占位提示。
- 调试请求传递文本、图片和附件；画布严格按后端事件高亮实际执行路径。

## 安全边界

- 表达式不开放方法调用或类型引用。
- Python 执行沿用现有超时和隔离能力。
- 文档解析限制 MIME、大小和解析时长，使用成熟解析器。
- 所有工具仍受应用授权列表约束。
